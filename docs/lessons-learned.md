# Lessons Learned

## Technical Lessons

- Redundancy should be designed at both Layer 2 and Layer 3.
- Firewall policy design should be based on traffic direction and required services, not only IP addresses.
- Centralized DHCP is easier to manage when DHCP Relay is correctly configured.
- FortiGate at the edge provides better visibility than router-only NAT designs.
- Documentation is as important as configuration in enterprise network projects.

## Portfolio Lessons

- A GitHub repository should not only contain configuration files.
- Good documentation shows design thinking and troubleshooting ability.
- Sanitized configs are safer and more professional for public sharing.
- Diagrams and validation checklists make the project easier to understand for recruiters and technical reviewers.
