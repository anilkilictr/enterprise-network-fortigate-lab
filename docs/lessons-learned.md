# Lessons Learned

## Technical Lessons

- A clear IP addressing plan prevents configuration drift.
- VLAN documentation is critical in multi-switch topologies.
- Routed distribution links make path validation easier.
- DHCP Relay simplifies centralized IP address management.
- Firewall policy design should follow least privilege principles.
- FortiGate at the edge provides better traffic visibility than router-only NAT designs.

## Portfolio Lessons

- A GitHub repository should not only contain configuration files.
- Good documentation shows design thinking and troubleshooting ability.
- Sanitized configs are safer and more professional for public sharing.
- Diagrams and validation checklists make the project easier to understand for technical reviewers.
